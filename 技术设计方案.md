# 股票智能分析系统 - 技术设计方案

## 📋 系统架构概览

### 整体架构
```
前端 (Vue.js) ← HTTP/REST API → 后端 (Python Flask/FastAPI) ← 数据层 (PostgreSQL + 缓存)

```

## 🗄️ 数据库设计

### 1. 股票基础信息表 (stocks)
```sql
CREATE TABLE stocks (
    id SERIAL PRIMARY KEY,
    stock_code VARCHAR(20) NOT NULL UNIQUE,      -- 股票代码
    stock_name VARCHAR(100) NOT NULL,           -- 股票名称
    market VARCHAR(10),                         -- 市场板块
    industry VARCHAR(100),                      -- 行业分类
    listing_date DATE,                          -- 上市日期
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);
```

### 2. 股票日线数据表 (stock_daily)
```sql
CREATE TABLE stock_daily (
    id SERIAL PRIMARY KEY,
    stock_code VARCHAR(20) NOT NULL,
    trade_date DATE NOT NULL,                   -- 交易日期
    open_price DECIMAL(10,4),                   -- 开盘价
    close_price DECIMAL(10,4),                  -- 收盘价
    high_price DECIMAL(10,4),                   -- 最高价
    low_price DECIMAL(10,4),                    -- 最低价
    volume BIGINT,                             -- 成交量
    turnover DECIMAL(15,2),                     -- 成交额
    pe_ratio DECIMAL(10,4),                    -- 市盈率
    pb_ratio DECIMAL(10,4),                    -- 市净率
    created_at TIMESTAMP DEFAULT NOW(),
    UNIQUE(stock_code, trade_date)
);
```

### 3. 策略配置表 (strategies)
```sql
CREATE TABLE strategies (
    id SERIAL PRIMARY KEY,
    name VARCHAR(100) NOT NULL,                 -- 策略名称
    type VARCHAR(20) NOT NULL,                  -- 策略类型: short/mid/long
    description TEXT,                           -- 策略描述
    parameters JSONB,                           -- 策略参数配置
    is_active BOOLEAN DEFAULT TRUE,            -- 是否启用
    created_at TIMESTAMP DEFAULT NOW()
);
```

### 4. 策略执行结果表 (strategy_results)
```sql
CREATE TABLE strategy_results (
    id SERIAL PRIMARY KEY,
    strategy_id INTEGER REFERENCES strategies(id),
    stock_code VARCHAR(20) NOT NULL,
    trade_date DATE NOT NULL,                   -- 执行日期
    signal_type VARCHAR(20),                   -- 信号类型: buy/sell/hold
    buy_price DECIMAL(10,4),                   -- 建议买入价
    stop_loss DECIMAL(10,4),                   -- 止损价
    take_profit DECIMAL(10,4),                 -- 止盈价
    confidence_score DECIMAL(5,4),             -- 置信度评分
    reasoning TEXT,                            -- 选股逻辑说明
    created_at TIMESTAMP DEFAULT NOW(),
    UNIQUE(strategy_id, stock_code, trade_date)
);
```

### 5. 数据更新记录表 (data_updates)
```sql
CREATE TABLE data_updates (
    id SERIAL PRIMARY KEY,
    update_type VARCHAR(50) NOT NULL,           -- 更新类型: daily/stock_info/history
    update_date DATE NOT NULL,                  -- 更新日期
    status VARCHAR(20) DEFAULT 'pending',       -- 状态: pending/running/completed/failed
    records_updated INTEGER DEFAULT 0,         -- 更新记录数
    error_message TEXT,                         -- 错误信息
    started_at TIMESTAMP,
    completed_at TIMESTAMP,
    created_at TIMESTAMP DEFAULT NOW()
);
```

## 🔌 API接口设计

### 1. 股票相关接口

#### GET /api/stocks - 获取股票列表
```javascript
// 请求参数
{
  "page": 1,
  "page_size": 20,
  "search": "",           // 股票代码或名称搜索
  "market": "",           // 市场筛选
  "industry": "",         // 行业筛选
  "sort_by": "code",      // 排序字段
  "sort_order": "asc"     // 排序方向
}

// 响应格式
{
  "data": [
    {
      "stock_code": "000001",
      "stock_name": "平安银行", 
      "market": "SZ",
      "industry": "银行",
      "latest_price": 15.20,
      "change_percent": 1.25
    }
  ],
  "total": 100,
  "page": 1,
  "page_size": 20
}
```

#### GET /api/stocks/{code}/daily - 获取股票日线数据
```javascript
// 请求参数
{
  "start_date": "2024-01-01",
  "end_date": "2024-12-31",
  "fields": "open,close,high,low,volume"  // 可选字段
}
```

### 2. 策略相关接口

#### POST /api/strategies/execute - 执行策略
```javascript
// 请求参数
{
  "strategy_ids": [1, 2, 3],     // 策略ID列表，空数组表示执行所有
  "trade_date": "2024-12-02"     // 执行日期，默认今天
}

// 响应格式
{
  "job_id": "strategy_123",
  "status": "started",
  "estimated_completion": "2024-12-02T18:00:00"
}
```

#### GET /api/strategies/results - 获取策略结果
```javascript
// 请求参数
{
  "strategy_id": 1,              // 可选，特定策略
  "trade_date": "2024-12-02",    // 可选，特定日期
  "stock_code": "000001",        // 可选，特定股票
  "page": 1,
  "page_size": 20
}
```

#### POST /api/strategies/{id}/rerun - 重跑单个策略
```javascript
// 响应格式
{
  "success": true,
  "message": "策略重跑已开始",
  "job_id": "strategy_1_rerun_123"
}
```

#### GET /api/system/status - 系统状态

## 🛡️ 错误处理与监控

### 错误处理策略
- **数据源错误**: 自动切换备用数据源，记录失败日志
- **策略执行错误**: 单个策略失败不影响其他策略，支持单独重跑
- **数据库错误**: 连接重试机制，事务回滚
- **网络错误**: 指数退避重试，超时控制

### 监控指标
- 数据更新成功率
- 策略执行成功率  
- API响应时间
- 系统资源使用率
